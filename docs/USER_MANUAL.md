## 1. 처음 시작하기 (5분)

### 1.1 카카오 연결
1. `/login` 접속
2. 노란 **"카카오로 연결하기"** 버튼 클릭
3. 카카오 권한 동의 (필수: `talk_message` "나에게 보내기")
4. "연결 완료 ✓" 메시지 확인

### 1.2 첫 종목 등록
1. `/dashboard` 접속
2. "종목 추가" 폼에서:
   - **종목 검색**: 이름(예: "삼성전자") 또는 티커(예: "005930") 입력
   - 자동완성 목록에서 클릭 선택
   - **평균단가** + **수량** 입력 (필수)
   - **손절선** + **익절선** 입력 (선택, 강력 권장)
3. **추가** 버튼

> 💡 손절·익절선을 설정해야 카카오 알림이 옵니다.

### 1.3 첫 분석
1. 보유 종목 클릭 → 종목 상세 페이지
2. **다시 분석** 버튼 클릭 → Claude 가 1~2초 후 분석 결과 표시
3. 결과: 보유 권고 / 부분매수 / 부분매도 / 전량매도 + 손절·익절 권장가

## 2. 화면별 가이드

### 2.1 홈 `/`
랜딩 페이지. 앱 소개와 빠른 진입.

| 섹션 | 내용 |
|---|---|
| **Hero** | 앱 슬로건 + "내 포트폴리오 열기" / "스크리너 결과 보기" |
| **주요 기능** | 4개 운영 중 + 2개 예정 표시 (포트폴리오, 스크리너, AI 분석, 카탈로그) |
| **빠른 분석** | 삼성전자 · SK하이닉스 · 카카오 · KODEX 200 · KODEX 코스닥150 즉시 분석 |
| **세팅 현황** | Supabase / Claude API / KIS / 카카오 / DART 연결 상태 |

### 2.2 대시보드 `/dashboard`

#### 상단 액션
- **← 홈 / 스크리너 / 카카오 연결** 버튼

#### 손절/익절 긴급 경고 배너 (있을 때만)
보유 종목 중 손절선/익절선 ±3% 이내 또는 도달 시 빨간 박스로 강조

#### 포트폴리오 요약 (보유 종목 있을 때)
- **총 평가금액** (KIS 실시간 시세 기준)
- **원가** · **미실현 손익** · **총 수익률**
- **오늘 수익률 (가중)** — 평가금액 가중평균 일간 변동
- **오늘 벤치마크 비교** — KODEX 200 / KODEX 코스닥150 대비

#### 종목 추가 폼
종목 검색 자동완성 + 평단/수량/손절/익절 입력 → **추가**

#### 보유 종목 목록
각 종목 카드:
- **좌**: 종목명 / 티커 / AI 추천 배지 ("보유" / "부분매수" / ...)
- **부 정보**: 현재가 · 등락률 · 평단 · 수량 · 평가금액 · 손절 · 익절 · 마지막 분석일
- **우**: 미실현 손익 + 수익률 (등락 색상)
- **버튼**: 분석 (상세 이동) / 삭제

### 2.3 종목 상세 `/holdings/[ticker]`
- 헤더: 큰 종목명 + 시장 배지 (KOSPI/KOSDAQ) + 티커
- **시세 카드** (QuoteCard) — KIS 실시간
- **AI 분석 패널** (AnalyzePanel):
  - **다시 분석** 버튼 → Claude Haiku 4.5 호출
  - **카카오톡으로 보내기** 버튼 → 분석 결과 자기 카톡으로
  - 결과 표시: 권고 · 확신도 · 핵심 근거 · 실행계획 (오늘 할 일/손절/익절/재점검) · 리스크
  - 캐시: 동일 marketData 라면 1시간 내 재호출 시 캐시 사용 (비용 절약)

### 2.4 스크리너 `/screener`
**주 2회(월·목 16:00 KST)** 자동 실행 결과 표시.

#### 상단
- 제목 + 설명 + **← 대시보드** 버튼

#### 안내 배너
"참고용 리스트, 매수 권유 아님" 디스클레이머

#### 실행 메타 (Run Meta)
- 최근 실행 시각
- 유니버스 N개 → 필터 M개 → 추림 3개
- 모델 (claude-haiku-4-5-...) + 비용 (원)

#### Pick 카드 3개
각각:
- **순위** + **종목명** + 티커 + 확신도 배지
- **Thesis** — 선정 근거 3줄 이내
- **진입 참고 / 손절선 / 익절선** — 정수 원 단위
- **주의해야 할 리스크** — 2~3개
- 버튼: **상세 분석** (종목 상세 페이지로)

### 2.5 로그인 `/login`
카카오 OAuth 연결 페이지.

| 상태 | 표시 |
|---|---|
| 미연결 | 노란 "카카오로 연결하기" 버튼 |

## 3. 자동 작동 — 카카오 알림

### 3.1 발송 트리거
**Vercel Cron 이 평일 KST 09:00~15:30 30분마다** 다음을 체크:

| 상황 | 알림 종류 |
|---|---|
| 현재가 ≤ 손절선 | `hit_stop` "[손절 도달]" |
| 손절선 +3% 이내 근접 | `near_stop` "[손절 근접]" |
| 현재가 ≥ 익절선 | `hit_take` "[익절 도달]" |
| 익절선 -3% 이내 근접 | `near_take` "[익절 근접]" |

### 3.2 카카오톡 메시지 예시
```
[손절 근접] 삼성에스디에스 (018260)

현재가 178,500원 (-1.2%)
손절선 175,000원
익절선 195,000원

손절선 근접 — 포지션 점검 후 대응 준비하세요.

※ 투자 참고용. 최종 판단은 직접.
```

### 3.3 중복 방지
- 동일 (종목, 알림 종류, 날짜) 조합은 **하루 1회만** 발송
- 발송 실패 시 다음 cron 에서 재시도

## 4. 일상 사용 시나리오

1. 종목 매수 직후 `/dashboard` 에서 **종목 추가** (손절·익절선 필수)
2. 이후 별 다른 작업 X — Vercel cron 이 자동으로 모니터링
3. 손절·익절 근접 시 카카오톡 자동 수신
4. 알림 수신 → 직접 매매 결정

### 시나리오 B: 새 종목 발굴
1. 월/목 16:30 경 `/screener` 접속

3. **다시 분석** → 추가 의견 확인
4. 매수 결정 시 `/dashboard` → **종목 추가** + 손절·익절선 설정

### 시나리오 C: 빠른 종목 분석
1. 홈 `/` → **빠른 분석** 섹션
2. 삼성전자 / SK하이닉스 / 카카오 / KODEX 200 / KODEX 코스닥150 클릭

## 5. AI 분석 결과 읽기

### 5.1 권고 (recommendation)
| 값 | 의미 |
|---|---|
| `hold` | 보유 유지 |
| `partial_buy` | 일부 추가매수 (몰빵 금지) |
| `partial_sell` | 일부 매도 |
| `full_sell` | 전량 매도 |

> **"잃지 않는 투자" 원칙**: 가능한 한 부분 매매 권장, 전량은 신중

### 5.2 확신도 (confidence)
- **high**: 명확한 신호 (수급·차트·펀더 일치)
- **medium**: 혼재된 신호 (사용자 판단 영역 큼)
- **low**: 불확실, 보수적 접근 권장

### 5.3 실행 계획 (action_plan)
- **immediate**: 오늘 할 행동
- **stop_loss**: 권장 손절선 (원 단위, 또는 null)
- **take_profit**: 권장 익절선 (원 단위, 또는 null)
- **review_at**: 다음 점검 시점 ("3거래일 후" 같이)

### 5.4 리스크 (risks)
주의해야 할 시나리오 2~3개. 이 중 하나라도 발생하면 손절선 재검토 필요.

## 6. 트러블슈팅

### 카카오 알림 안 옴
1. `/login` 접속 → "현재 연결됨" 표시 여부 확인
2. 안 됐으면 카카오 재연결
3. 권한 동의 시 `talk_message` 체크 필수
4. 카카오톡 자체에서 메시지 수신 거부 설정 안 했는지 확인

### "다시 분석" 클릭 후 응답 없음
- 1~2초 정도 정상. 5초 이상 무응답이면 잠시 후 재시도
- 캐시 적용 시 (1시간 내 동일 시세) 즉시 응답
- ANTHROPIC_API_KEY 만료 시 에러 표시

### 스크리너에 아무 결과 없음
- "아직 실행 기록이 없습니다" — 워크플로우 첫 실행 대기 (월/목 16:00)
- "최근 실행 실패" — `screener_runs.error_message` 확인 필요

### 종목 추가 후 종목명이 티커로만 표시
- `stocks` 테이블에 해당 티커 카탈로그 row 없는 경우
- 월 1회 KRX 카탈로그 자동 적재됨, 또는 GitHub Actions → load-stocks 수동 실행

### 시세가 0 또는 안 보임
- KIS API 호출 실패 — KIS_APP_KEY/SECRET 유효성 확인
- 장 시간 외엔 prev_close 만 표시될 수 있음

## 7. 자주 묻는 질문

### Q1. 실제 매수/매도가 자동으로 일어나나요?
**아니요.** 이 앱은 분석·알림 도구이지 자동 매매 시스템이 아닙니다. KIS 주문 API 는 의도적으로 미구현. 매매는 본인이 직접 증권사 앱에서 수행.

### Q2. 카카오 알림이 다른 사람한테 가나요?
**아니요.** "나에게 보내기" 권한만 사용. 본인 카카오톡에만 전송됩니다.

### Q3. 비용은 얼마나 드나요?
- Claude API: 월 100~500원 (Haiku 4.5 기준)
- KIS API: 무료
- 카카오: 무료 (1일 60건 한도, 실제 사용량은 훨씬 적음)
- Supabase / Vercel: Free Tier 내

### Q4. 데이터는 안전한가요?
- 모든 환경변수 (API 키, 토큰) Vercel/Supabase 비밀 저장소
- RLS 적용 (service_role 만 접근)
- 사용자 데이터는 보유 종목, 손절/익절선, 카카오 토큰만 저장
- 매매 권한 없음

### Q5. 스크리너 pick 을 따라 사면 수익이 나나요?
**보장 없음.** 스크리너는 **참고용 리스트**이지 매수 신호가 아닙니다. 알고리즘 품질은 데이터 누적 후 검증 필요.

### Q6. AI 가 틀린 분석을 했을 때
- 모든 응답은 **참고용**
- 확신도 `high` 라도 시장은 예측 불가
- 시스템에 내장된 손절선이 최후 안전망 — 무조건 지킬 것

## 8. 핵심 원칙 (시스템에 내장됨)

1. **잃지 않는 투자** — 원금 보호 > 수익 추구
2. **부분 매매** — 전량 매수/매도 지양
3. **구체 가격** — 손절선·익절선 반드시 명시
4. **확신도 명시** — high/medium/low 항상 표기
5. **참고용 명시** — 매수 권유 아님 디스클레이머

---

## 9. 데이터 표기 관례

- **상승 = 빨강, 하락 = 파랑** (한국 시장 관례)
- 가격은 **원 단위 정수**, 천 단위 구분 쉼표
- 수익률은 **소수 둘째 자리** + % 기호
- 시가총액은 한글 표기 ("1.29조원", "12,900억원")
- 시각은 **KST 기준**, 한국어 포맷
